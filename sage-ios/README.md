# Sage iOS App

Native SwiftUI iOS app for the Sage AI Financial Assistant, sharing the React frontend's architecture and design patterns.

## 🚀 Quick Start

### Prerequisites
- **Xcode 16+**
- **iOS 16.0+** deployment target
- **Swift 5.9+**
- **Ruby** (for XcodeGen)

### Setup

```bash
# Install XcodeGen for project generation
brew install xcodegen

# Navigate to project directory
cd sage-ios

# Generate Xcode project from project.yml
xcodegen generate

# Open in Xcode
open Sage.xcodeproj

# Select iPhone simulator and press ▶️ to run
```

## 🔐 Authentication

### Supported Methods
1. **Email/Password**: Through Supabase
2. **Google OAuth**: Deep-link to native auth (WebView fallback)

### Flow
```
User Login
  ↓
Supabase Auth (email/password or OAuth)
  ↓
JWT token stored in Supabase session
  ↓
ChatViewModel retrieves JWT via AuthService
  ↓
API requests include "Authorization: Bearer <JWT>"
```

## 💬 Chat Features

### Message Streaming
- **SSE Protocol**: Server-Sent Events for real-time responses
- **Event Types**: text, toolUse, plan, error, done
- **Markdown Rendering**: Using MarkdownUI framework
- **Action Bar**: Copy, thumbs up/down, retry buttons (desktop-like)

### Supported Models
All default providers from desktop app (8 total):
- OpenRouter, MiniMax, Z.ai, Volcengine, 302.AI, Ollama, SiliconFlow, Kimi

## 🐛 Known Issues

1. **OAuth**: Google OAuth native deep-link flow still needs end-to-end verification
   - **Workaround**: Use email/password login
   - **Status**: Native SwiftUI client no longer uses a WebView shell

2. **API Token**: Currently hardcoded in APIClient.swift
   - **Fix**: Move to Keychain or environment-based injection
   - **Security**: For local testing only

3. **Markdown Rendering**: Limited table support
   - **Workaround**: Use code blocks for tabular data
   - **Enhancement**: Consider custom table component
