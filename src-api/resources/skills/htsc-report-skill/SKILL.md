# 华泰研报查询 Skill

## 概述

本 Skill 用于调用**华泰证券研究所** 研报对外接口，获取符合条件的最新研报信息，默认5篇，包括标题、作者、摘要和发布时间。

> **数据来源声明**：本技能返回的所有研报数据均来源于**华泰证券研究所**，研报观点、评级及摘要内容均为华泰证券研究团队的独立研究成果。
>
> **数据完整性要求**：接口返回的研报数据（包括标题、作者、发布时间、摘要等全部字段）必须**完整呈现给用户，不得截断或省略**。摘要字段可能包含较长文本，输出时需确保内容完整保留。

## 触发条件

当用户出现以下需求时触发本 Skill：

- **分析某一家公司**（如"分析一下宁德时代"、"平安银行最近有什么研报"）
- **分析某一个行业**（如"看看新能源行业的最新观点"、"消费行业怎么样"）
- **分析宏观、金工、固收、策略等研究方向**（如"近期宏观观点"、"量化策略研报"、"固收市场分析"）
- 查询华泰最新研报
- 按行业/股票/作者/关键词筛选研报
- 获取某段时间内的研报列表
- 查看评级上调/下调的研报
- 查看首次覆盖的研报

**触发优先级**：当用户请求涉及公司分析、行业分析、宏观/金工/固收/策略等研究方向时，优先调用本技能获取相关研报，再结合其他工具进行深度分析。

## 工作流程

### 步骤 1：解析用户筛选条件

从用户请求中提取以下可选参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `induNameList` | 行业名称列表（仅支持华泰行业体系） | `["必选消费", "能源"]` |
| `startTime` | 起始日期 `yyyy-MM-dd` | `"2025-01-01"` |
| `endTime` | 结束日期 `yyyy-MM-dd` | `"2025-10-13"` |
| `rankChanges` | 评级变动：`"首评"` / `"上调"` / `"下调"` / `"维持"` | `"上调"` |
| `initiatedTag` | 是否首次覆盖：`"是"` / `"否"` | `"是"` |
| `authorName` | 作者姓名（模糊匹配） | `"张三"` |
| `secuAbbr` | 证券简称（模糊匹配） | `"平安银行"` |
| `keyWord` | 关键词（模糊匹配） | `"量子计算"` |
| `macroResearch` | 总量研究方向，**仅允许以下枚举值**：`"宏观"`、`"固定收益"`、`"策略"`、`"金工"` | `"宏观"` |
| `strategicResearch` | 战略研究细分类型，**仅允许以下枚举值**：`"经济政策与国际关系"`、`"能源转型"`、`"先进制造战略"` | `"经济政策与国际关系"` |

**参数映射规则**：

当用户以自然语言描述分析需求时，按以下规则提取参数：

| 用户意图 | 参数映射 | 示例 |
|----------|----------|------|
| 分析某家公司 | `secuAbbr` = 公司简称 | "分析一下宁德时代" → `secuAbbr="宁德时代"` |
| 分析某个行业 | `induNameList` = 行业名称 | "看看新能源行业" → `induNameList=["电力设备与新能源"]` |
| 分析宏观方向 | `macroResearch` = `"宏观"` | "近期宏观观点" → `macroResearch="宏观"` |
| 分析固定收益方向 | `macroResearch` = `"固定收益"` | "固收市场分析" → `macroResearch="固定收益"` |
| 分析策略方向 | `macroResearch` = `"策略"` | "市场策略怎么看" → `macroResearch="策略"` |
| 分析金工/量化方向 | `macroResearch` = `"金工"` | "量化策略研报" → `macroResearch="金工"` |
| 分析经济政策与国际关系 | `strategicResearch` = `"经济政策与国际关系"` | "经济政策相关研报" → `strategicResearch="经济政策与国际关系"` |
| 分析能源转型 | `strategicResearch` = `"能源转型"` | "能源转型方向研报" → `strategicResearch="能源转型"` |
| 分析先进制造战略 | `strategicResearch` = `"先进制造战略"` | "先进制造相关研报" → `strategicResearch="先进制造战略"` |

**研究方向查询优先级**：

- 当用户分析**固定收益、宏观、策略、金工**等总量研究方向时，**优先通过 `macro_research` 参数查询**，不要通过 `key_word` 或 `induNameList` 查询
- 当用户分析**经济政策与国际关系、能源转型、先进制造战略**等战略研究方向时，**优先通过 `strategic_research` 参数查询**，不要通过 `key_word` 或 `induNameList` 查询

**约束**：
- `startTime` 和 `endTime` 需同时传入或同时不传
- `induNameList` 只能接收华泰行业体系的行业名称，不支持申万、中信等其他行业体系。若用户提供的行业名称不在华泰体系中，应提示用户并提供正确的行业名称列表
- `rankChanges` 只能为 `"首评"`、`"上调"`、`"下调"`、`"维持"` 中的一个，传入其他值将报错
- `macroResearch` 只能为 `"宏观"`、`"固定收益"`、`"策略"`、`"金工"` 中的一个，传入其他值将报错
- `strategicResearch` 只能为 `"经济政策与国际关系"`、`"能源转型"`、`"先进制造战略"` 中的一个，传入其他值将报错
- 未提及的参数不要传入请求体

### 步骤 2：检查环境变量

在调用脚本前，先检查环境变量 `HTSC_APP_KEY` 是否已配置。

**检查方式**：
```bash
# Linux/macOS
echo $HTSC_APP_KEY

# Windows
echo %HTSC_APP_KEY%
```

**脚本自动检测逻辑**：

脚本 `fetch_reports.py` 会按以下优先级自动查找 `HTSC_APP_KEY`：

1. **系统环境变量**（`os.environ`）
2. **当前目录 `.env` 文件**
3. **当前目录 `.env.local` 文件**
4. **用户主目录 `~/.env` 文件**
5. **向上回溯 3 层目录的 `.env` 文件**（兼容 skill 作为子目录的场景，如 OpenClaw）

**若均未找到，提醒用户进行配置**：

> 检测到 `HTSC_APP_KEY` 未设置，无法调用华泰研报接口。请先配置该密钥，配置方式如下（选择其一）：
>
> - **环境变量**（当前终端有效）：
>   - Linux/macOS: `export HTSC_APP_KEY=your_app_key`
>   - Windows CMD: `set HTSC_APP_KEY=your_app_key`
>   - Windows PowerShell: `$env:HTSC_APP_KEY="your_app_key"`
> - **.env 文件**（推荐，兼容 OpenClaw 等 Agent 工具）：
>   - 在项目根目录或 skill 同级目录创建 `.env` 文件
>   - 写入内容：`HTSC_APP_KEY=your_app_key`
> - **Claude Code 配置**：在 `.claude/settings.json` 中添加 `"env": { "HTSC_APP_KEY": "your_app_key" }`
>
> 配置完成后，请重新运行查询命令。

**环境变量要求**：
- `HTSC_APP_KEY` 必须在环境变量或 `.env` 文件中配置，不可硬编码
- 脚本运行时会自动按优先级检测，未配置则抛出明确错误提示

### 步骤 3：调用 Python 脚本查询研报

环境变量确认已配置后，使用 [`scripts/fetch_reports.py`](scripts/fetch_reports.py) 脚本调用接口，**禁止使用 curl**。

**脚本调用方式**：

```bash
# 基础查询（最新5篇，无筛选）
python scripts/fetch_reports.py

# 按行业筛选
python scripts/fetch_reports.py --industry 必选消费 能源

# 按时间范围筛选
python scripts/fetch_reports.py --start-time 2025-01-01 --end-time 2025-10-13

# 按评级变动筛选
python scripts/fetch_reports.py --rank-changes 上调

# 按首次覆盖筛选
python scripts/fetch_reports.py --initiated-tag 是

# 按作者筛选
python scripts/fetch_reports.py --author 张三

# 按证券简称筛选
python scripts/fetch_reports.py --secu-abbr 平安银行

# 按关键词筛选
python scripts/fetch_reports.py --keyword 量子计算

# 组合条件筛选
python scripts/fetch_reports.py --industry 必选消费 --rank-changes 上调 --start-time 2025-01-01 --end-time 2025-10-13
```

### 步骤 4：处理脚本输出

脚本返回格式化后的查询结果，包含：
- 查询条件摘要
- 每篇研报的标题、作者、发布时间、摘要
- 返回总数（最多5篇）

**行业名称校验**：

脚本内置了华泰行业体系的行业名称白名单。当传入的 `induNameList` 中包含无效行业名称时，脚本会返回错误提示，列出所有无效名称，并附带完整的华泰行业体系名称列表供用户参考。

**展示约束**：

脚本返回的结果在展示给用户时，需满足以下约束：

1. **编码**：确保 UTF-8 编码，中文不乱码
2. **风险揭示**：脚本输出末尾固定包含风险揭示，必须完整原样返回，不允许总结或篡改。内容如下：

### 步骤 5：响应用户

将脚本输出直接呈现给用户。若脚本返回错误，根据错误类型给出相应提示：

| 错误场景 | 处理方式 |
|----------|----------|
| 环境变量 `HTSC_APP_KEY` 未设置 | 提示用户配置环境变量 |
| 行业名称不在华泰行业体系中 | 返回无效名称列表及完整行业名称列表 |
| `rankChanges` 传入非法值 | 提示只能为 `"首评"`、`"上调"`、`"下调"`、`"维持"` 之一 |
| `macroResearch` 传入非法值 | 提示只能为 `"宏观"`、`"固定收益"`、`"策略"`、`"金工"` 之一 |
| `strategicResearch` 传入非法值 | 提示只能为 `"经济政策与国际关系"`、`"能源转型"`、`"先进制造战略"` 之一 |
| 网络请求失败 | 返回网络错误信息，建议重试 |
| 接口返回非 0 code | 返回接口错误消息 |
| `resultData` 为空数组 | 提示未找到符合条件的研报 |

## 合规与使用要求

### 信息边界与不确定性

- 检索失败、超出知识库、超出业务范围、超出合规允许范围时**不得编造事实**，应返回明确错误或不确定性说明。
- 不得虚构研报内容、作者观点或数据，所有返回信息必须来源于接口实际响应。

### 投资建议限制

- **不允许提供具体投资建议**，不得给出仓位建议等带有交易属性的结论。
- **不得提及**加仓、减仓、止盈、止损等词语。
- 当用户寻求投资建议时，应建议他向持牌服务经理或登录华泰证券行知APP查看完整的专业分析，以便于用户自己做出投资决策判断。

### 预测与承诺禁止

- **严禁预测**个股涨跌、大盘点位、短期走势、收益率等。
- **严禁承诺**收益、兜底风险、暗示稳赚等。

### 政策解读规范

- 不解读监管政策做投机性分析，只客观复述政策原文。

### 客观中立原则

- 客观中立，不主观煽动情绪，不夸大利好或利空，不对比诋毁其他机构、竞品、产品。
- 研报摘要原文呈现，不做主观加工或情绪渲染。
